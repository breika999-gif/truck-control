ЗАДАЧА 3: ТАХОГРАФ МОДУЛ (Bluetooth + EU 561/2006)
------------------------------------------
Устройства: VDO DTCO 4.1 / Stoneridge SE5000 (ново поколение с BLE)
Протокол: ISO 16844-7 ITS BLE Interface

Готов код в src/features/tacho/:
  - TachoBleService.ts   → scan + connect + subscribe (ACTIVITY, HOS, SPEED)
  - VdoItsProtocol.ts    → BLE UUID-а, байт декодер
  - TachoParser.ts       → HEX → активност + Gemini формат
  - useTachoBluetooth.ts → React hook обвивка (с BLE permissions)
  - screens/TachoScreen.tsx → UI

СТАТУС: [✅] ГОТОВО

----------------------------------------------------------------

ЗАДАЧА 4: GEMINI ЧЕТЕ ТАХОГРАФА
------------------------------------------
- /api/tacho/live_update endpoint в backend
- Gemini получава live BLE данни в system prompt
- Проактивни предупреждения при < 30 мин до лимит
СТАТУС: [✅] ГОТОВО

ЗАДАЧА 5: GEMINI — УМЕН АСИСТЕНТ ЗА ШОФЬОРА
------------------------------------------
5А. ТАХО + МАРШРУТ — може ли да стигна?
  - Gemini знае текущото тахо (непрекъснато, дневно, седмично)
  - При въпрос "Мога ли да стигна до Бургас?" → изчислява разстояние,
    прибл. часове, сравнява с оставащото шофьорско време
  - Ако не може → казва кога трябва да почине и предлага спирка
  СТАТУС: [ ] ЧАКА

5Б. TRANSPARKING — отваря Chrome за конкретен паркинг
  - При въпрос за конкретен паркинг от картата → отваря Chrome с
    URL: https://truckerapps.eu/transparking/pl/map/
    (или конкретна страница на паркинга ако имаме ID)
  - Потребителят сам чете и казва на Gemini данните
  - Gemini анализира: има ли душове, места, цена и т.н.
  СТАТУС: [ ] ЧАКА
